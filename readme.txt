
本apkは制作中のAndroidアプリケーション「viva 2D Studio」を職務経歴向けに少し改造したものです。
完成までに後1年ほどを予定しています。

＜概要＞
現代の一番メジャーな再生コンテンツは「動画」ですが、動画ファイルではない、新しいコンテンツ再生用のフォーマット(*.vivaファイル)	
を作っています。																														
動画は、そのファイル自体に「映像と音を再生する」機能しか持ちませんが、比較して以下の特徴を持ちます。	

＜特徴＞

1) OpenGLベースのアニメーションを重畳表示	

vivaファイルでは画面に表示する全てのキャラクタ単位で「座標」「テクスチャ」などの情報を保持しています。
これらのキャラクタは同じ時間軸上で再生されます。処理落ちしても再生時間がズレない設計のため、動画の再生のように
指定した時間の画面を即座に表示することができます。
動画の再生に合わせて上にvivaファイルの再生データを同期して重畳表示することも可能です。
また、OpenGLなので「ボタン」のキャラクタを画面内に設置して、ユーザーがクリックしたらアクションする。
といったユーザー視点のインタフェイスも提供することができます。

2) 他人の作ったデータを共有できる	
作成したキャラクタデータは外部公開することにより共有が可能です。	
例えば、あるクリエイターが「流れ星」が降るエフェクトを作成して外部公開したとします。	
これを自分の作成するプロジェクトに取り込めば、そのエフェクトが自分のコンテンツですぐに再生できるようになります。	
クリエイターはコンテンツを作成する時間を大幅に節約することができます。	


3) データ容量が動画に比べて非常に少ない	

動画のデータ容量は再生時間に比例しますがvivaファイルではキャラクタ数(+画像や動画ファイル)に比例します。
例えば画像も動画も使わないvivaファイルは再生時間に関わらず20kb程度のデータ容量しか持ちません。
これによりスピーディな再生開始が可能となります。

4) コンテンツ制作に必要なツールは全てアプリ内に内包	

vivaファイルの作成に必要なツールは基本的に本アプリのみです。これによりクリエイターの敷居をかなり
低い位置まで下げることができます。
ただし、画像のトリミングや動画のカットなどの基本的な部分は別アプリで行う必要があります。


5) 用途が多彩	
vivaファイルは動画と異なり、様々な用途で利用することができます。	
a) (広告配信サービス業者様向け) 今までの一枚絵、動画コンテンツからvivaファイルによるダイナミックなアニメーション広告	
b) (アプリベンダー向け) viva 2d Studio SDKを使用し、アプリ起動時にアニメーションする自社ロゴを表示	
c) (動画配信サービス向け) 動画再生の上にvivaファイルを重畳表示して同期再生	
	(niconico動画の流れるコメントがより表現力多彩になるイメージです)
d) 2Dゲーム専用SDKを作成すれば、ゲームプログラマーはopenGLの知識を必要とせずに	
	vivaファイルを読み込むだけで2Dキャラクタのアニメーションやエフェクトを実装することができます。
その他、「画面に何かを表示する」ことであれば様々な用途が考えられると思います。	


＜apk確認手順＞

1) アプリを起動します。
2) Debug Projectボタンを押下します。
3) 左サイドメニューからPreviewを選択します。
4) サンプルデータが再生されます。再生されているデータは動画ではなくOpenGLによる2Dテクスチャです。
5) 画面下のPauseボタンを押すと停止、タイムバーを動かすとその時間まで画面が移動します。
   (まるでyoutubeのように2D描画を再生、停止することができ,動画に比べて再生に必要なデータ容量が非常に少ないのが特徴です)
